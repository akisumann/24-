# 七年巫女神：AI向け引き継ぎ書

この文書は、別のAIへゲームの現状・設計意図・変更時の注意点を渡すためのものです。

## 1. プロジェクト

- リポジトリ：`akisumann/24-`
- 公開ページ：`https://akisumann.github.io/24-/`
- 形式：HTML/CSS/JavaScriptのみで動くブラウザゲーム
- 主題：国家と神の契約によって、常時50人の巫女を維持し、七年ごとの世代交代・神の娘・母系一族の繁栄を観察する長期シミュレーション

## 2. 重要な作業方針

1. 現在の固定基盤を勝手に全面改修しない。
2. 変更は可能な限り最小差分にする。
3. 試作・調整中の機能は、ユーザーが明示的に承認するまで固定版扱いにしない。
4. 既存機能を消したり、別設計へ置き換えたりする前に確認する。
5. コードを変更する前に、必ずGitHub上の最新版ファイルを取得する。
6. 変更後は、触ったファイルと変更内容を具体的に説明する。

## 3. 基本ゲーム

### 時間進行

- 1ターン＝7年。
- 6歳、13歳、20歳、27歳、34歳の五世代が存在する。
- 現在能力の発現率は年齢ごとに20%、40%、60%、80%、100%。
- 年齢上昇では最大能力そのものは変化しない。

### 巫女

- 現役巫女は常時50人。
- ステータスは以下の7項目。
  - HP
  - MP
  - ATK
  - DEF
  - INT
  - SPD
  - DEX
- 各巫女は既存の固有スキルを5個持つ。
- スキルは戦闘専用ではなく、儀礼、治療、制作、調査、移動、護衛なども含む。

### 七年の大儀

- 20歳、27歳、34歳の正式巫女から最大能力平均上位10人を選ぶ。
- 選ばれた巫女は神との娘を一人ずつ産む。
- 34歳の巫女は大儀後に任期終了。
- 空席は神の娘と国家公募採用者で補充し、50人を維持する。

### 神の娘

各ステータスを独立して次の式で決定する。

```text
round(((母親の該当最大能力 + 神レベル) / 2) × 0.50〜1.50)
```

- 神レベルは通常、現役巫女数と同じ50。
- 娘は母親と同じ姓になる。
- 各項目で別々の乱数を使う。

## 4. 一族と親類縁者

- 同じ姓は同じ一族として集計する。
- これは必ずしも近親家族を意味せず、千年単位では遠い支流も同じ氏族名に含まれる。
- 同じ姓なら、はるか昔に共通祖先がいた氏族としてまとめてよい。
- 現役巫女、元巫女、血縁者、配偶者などが増減する。
- 強い巫女が大儀で娘を残し続けると、同姓の現役巫女が増えて大氏族化する。
- 一方で公募出身者の大当たりによって新しい姓が急増し、古い巫女一族が現役ゼロになって断絶することもある。

## 5. 世界設定上の巫女制度

- 巫女は国家と神の契約に基づく特別な身分・資格である。
- 国家が維持費・教育・医療・生活を負担するため、巫女は国家へ実質的な貢献を行う。
- 長期的には、現役50人を中心に、元巫女、親族、技師、医師、官吏、警備、教育担当など数百人規模の国家機関・共同体へ発展する。
- 巫女は辞職可能であり、制度上は禁止されていない。
- ただし辞職によって一族の現役席、大儀選抜の可能性、将来の神の娘、長期的な氏族繁栄が減るため、心理的・社会的な圧力は存在する。
- この圧力は特別な残酷制度というより、重要な会社員、貴族後継者、国家職などが辞める際に生じる責任や引き継ぎの圧力を大きくしたものに近い。
- 貴族社会なので婚姻は現代的な完全自由恋愛ではなく、国家・巫女院・一族・相手家が政治、血統、土地、役職、子供の扱いなどを交渉する。

## 6. 現在のタイマン模擬戦試作

この機能は現在、試作状態であり、ユーザー承認前の固定版ではない。

### 選手選択

- 現役巫女50人から巫女Aと巫女Bを選ぶ。
- 同一人物同士は不可。
- 二人をランダム選択する機能もある。
- 対戦開始後は自動で3ターン進む観戦型。

### 基本判定

各ステータス値をダイスの最大値として使う。

```text
ATK 68 → 1d68
SPD 52 → 1d52
```

各ターン、選ばれた2能力のダイスを一回ずつ振り、合計値を比較する。

### 第1ターン：得意分野

- 各巫女の現在能力上位2項目を使用。

```text
1d上位1位 + 1d上位2位
```

### 第2ターン：対応力

- 第1ターンで使った上位2項目を除く。
- 残り5項目から各自ランダムに2項目を選ぶ。

```text
1dランダム能力1 + 1dランダム能力2
```

### 第3ターン：切り札

- 各自の最高能力を必ず使う。
- 残り6項目からランダムに1項目を選ぶ。

```text
1d最高能力 + 1dランダム能力
```

### 勝敗

- 各ターンの合計が高い側が1勝。
- 3ターン中2勝以上した側が総合勝者。
- 同点時は、同じ能力組み合わせのままそのターンだけ再ロールする。

### スキル補正

- 各巫女が元から持つ5スキルだけを使う。
- 新しい戦闘専用スキルを追加してはいけない。
- そのターンの攻防で自然に使えそうな既存スキルがある場合、ダイス合計を1.1倍する。
- 複数該当しても倍率は重ならず、1.1倍のみ。
- 非戦闘技能を無理に戦闘へ結びつけない。
- 現在のコードでは、戦闘利用可能とみなすスキル名を `battle.js` 内で暫定的に列挙している。
- この列挙は試作であり、ユーザー確認後に調整する。

### 戦闘描写

各ターンの文章は、主に次を使って構成する。

- 選ばれた2能力
- ダイス結果
- 使用可能だったスキル
- 勝敗
- 数値差

例：

```text
攻撃と素早さで勝利
→ 素早く相手を翻弄し、斬撃を通した。

次に防御と知識で敗北
→ 最初はテンポを取ったが、相手に攻め筋を読まれ、守勢を崩された。
```

数値差が小さければ接戦、大きければ圧倒として描く。

## 7. 主なファイル

- `index.html`
  - 画面構造。
  - 模擬戦UIもここに追加されている。
- `style.css`
  - 全体表示とスマホ対応。
- `data.js`
  - ステータス名、姓、名前、スキル候補、性格文など。
- `core.js`
  - 巫女生成、能力、神の娘、一族人口、役務配属など。
- `ui.js`
  - 七年進行、名簿、詳細、一族、履歴などの表示。
- `battle.js`
  - 現在のタイマン模擬戦試作。
- `batch.js`
  - 七十年進行。
- `hold.js`
  - 長押し関連の試作。
- `GAME_RULES.md`
  - ゲームルールの説明。
- `WORLD_SETTING.md`
  - 世界設定の説明。ただし一部は最新の会話内容へまだ完全追従していない可能性がある。

## 8. 現在の注意点

- 模擬戦は既存の七年進行や一族計算へ影響を与えない。
- 勝敗や戦績はまだ保存しない。
- 負傷、死亡、報酬、評判変動もまだない。
- スキル利用可否は暫定判定。
- 戦闘描写は能力の組み合わせを元にした短い定型文。
- 現在の戦闘機能はまずタイマンを完成させる段階であり、部隊戦へ勝手に拡張しない。
